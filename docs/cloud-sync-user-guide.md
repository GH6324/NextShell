# 云同步使用手册

本文面向 NextShell 使用者，说明当前版本的 workspace 云同步如何配置、如何协作，以及哪些数据会被同步。

## 1. 核心概念

### 1.1 Workspace Repo

每个 `workspace` 都是一套独立的共享资产仓库，包含：

- 服务器连接
- SSH 密钥
- 代理配置

这些资产会按 workspace 维度同步、记录历史，并支持冲突处理与恢复旧版本。

### 1.2 Workspace 命令集

每个 `workspace` 还带一套共享命令集。它和服务器资产分开同步：

- 命令集按整集覆盖同步
- 不做 commit 历史
- 最后一次成功写入的内容生效

### 1.3 本地资产

本地资产继续独立存在，不参与 workspace 云同步：

- 本地连接
- 本地 SSH 密钥
- 本地代理
- 本地命令

你可以同时使用：

- 多个 workspace 的共享资产
- 不参与同步的本地资产

它们互不冲突。

## 2. 会同步什么

### 2.1 会同步

- 连接基础信息：主机、端口、用户名、分组、标签、备注等
- 连接凭据：密码类认证的密文
- SSH 密钥：私钥内容与口令
- 代理：代理地址、用户名、密码
- Workspace 命令：名称、描述、分组、命令内容、是否模板

### 2.2 不会同步

- 本地命令库
- 命令模板参数的本地记忆值
- 命令历史
- 非 workspace 资产

## 3. 首次接入

在客户端打开 `设置` 中的 `Workspace Repos` 面板。

点击 `添加工作区` 后，填写：

- `API 地址`：服务端地址，例如 `https://sync.example.com`
- `工作区名称`：团队共享 workspace 名称
- `显示名称`：可选，仅本地显示更友好的名称
- `工作区密码`：该 workspace 的共享凭据
- `拉取间隔`：后台自动同步间隔，默认 300 秒
- `忽略 TLS 错误`：仅用于自签名证书调试环境

保存后客户端会立即：

- 建立本地 workspace 记录
- 拉取远端服务器资产和共享命令
- 将结果落到本地缓存

如果这是这个 workspace 第一次被任何人使用，服务端会自动创建它。

## 4. 日常使用

## 4.1 管理工作区

`Workspace Repos` 面板支持：

- `同步`：立即执行一次拉取/推送
- `编辑`：修改地址、名称、轮询间隔、是否启用等
- `复制 Token`：导出可分享的工作区配置
- `粘贴 Token`：从其他同事复制的 Token 自动填充
- `删除`：移除该 workspace 及其本地缓存

面板同时显示：

- 当前同步状态
- 本地和远端 head commit
- 最近几条历史
- 冲突数量和错误状态

## 4.2 新建共享连接

在连接管理器中新建连接时：

1. 在 `分组路径` 中把作用域切到 `工作区`
2. 选择 `所属 Workspace`
3. 填写连接信息并保存

保存后这个连接会进入对应 workspace 仓库，其他使用同一 workspace 的同事和设备可以拉取到它。

注意：

- 共享连接只能引用同一 workspace 下的 SSH 密钥和代理
- 已经属于某个 workspace 的共享连接，根节点会锁定，不能改到别的 workspace

## 4.3 新建共享 SSH 密钥和代理

在 SSH 密钥管理和代理管理面板中，都有 `保存位置` 选项：

- `本地`
- `工作区`

如果选择 `工作区`，还需要指定目标 workspace。保存后会作为共享资产同步出去。

## 4.4 使用共享命令

在命令中心顶部，可以切换：

- `本地`
- 各个 workspace

当你切到某个 workspace 时：

- 新建/编辑/删除命令会修改该 workspace 的共享命令集
- 其他同事拉取后会看到同一套命令

模板参数的本地记忆值不会上传，因此每个人的临时输入不会污染别人。

## 5. Token 分享

`Workspace Repos` 面板里的 `复制 Token` 会导出一段可分享字符串。它包含：

- API 地址
- workspace 名称
- 显示名称
- workspace 密码
- 拉取间隔
- TLS 配置

这意味着：

- Token 本身是敏感信息
- 应只在可信渠道中传递
- 收到 Token 的人粘贴后即可直接接入同一个 workspace

## 6. 冲突与历史

### 6.1 什么时候会冲突

当本地和云端同时修改了同一个资源时，客户端会进入冲突状态。

当前冲突处理面板提供两种选择：

- `保留本地`
- `接受云端`

无论选择哪种方式，旧版本都会进入回收站。

### 6.2 恢复历史

在 `Workspace Repos` 面板中，每个 workspace 都会显示最近的 commit 历史。点击某条记录右侧的 `恢复`，客户端会把该版本恢复为当前工作状态，然后再次同步。

适合处理：

- 误删连接
- 错误修改分组或凭据
- 回滚到一个已知可用版本

## 7. 删除工作区

在 `Workspace Repos` 中删除某个 workspace 时，会清除本机上的：

- 该 workspace 的连接、SSH 密钥、代理实体缓存
- 该 workspace 的命令缓存
- 该 workspace 的同步状态与冲突记录

不会影响：

- 本地连接
- 本地密钥
- 本地代理
- 本地命令
- 其他 workspace

## 8. 常见建议

- 生产环境请使用有效 HTTPS 证书，不要长期启用 `忽略 TLS 错误`
- 给不同团队使用不同 workspace，不要混用一个全局 workspace
- 共享 Token 前，确认接收方确实应该拥有该 workspace 的全部资产权限
- 大量资产调整后可以手动点一次 `同步`，不要只等轮询
- 对重要 workspace，建议定期备份服务端 SQLite 数据库

## 9. 常见问题

### 9.1 添加工作区时报 TLS 错误

优先检查服务端证书是否有效、域名是否匹配。只有在开发环境使用自签名证书时，才建议临时启用 `忽略 TLS 错误`。

### 9.2 共享连接保存时提示找不到 workspace

先到 `Workspace Repos` 中确认该 workspace 已添加且处于启用状态。

### 9.3 连接引用不了某个 SSH 密钥或代理

连接只能引用同一 workspace 下的共享 SSH 密钥 / 代理，或者引用本地作用域中的本地资产。不能跨 workspace 引用。

### 9.4 粘贴 Token 后还是连接不上

重点检查：

- `API 地址` 是否可达
- 证书是否可信
- Token 中的 `workspacePassword` 是否已过期或被改动

### 9.5 为什么同事看不到我本地记住的模板参数

模板参数记忆值是本地数据，不属于共享命令内容，设计上不会同步。
